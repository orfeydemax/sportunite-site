# Hero Section Creator

Скилл для создания качественных первых экранов сайтов с полным discovery-процессом.

## Workflow

### Шаг 1: Discovery (обязателен)

Прежде чем создавать код, ВСЕГДА задай следующие вопросы:

**Обязательные вопросы:**

1. **Бизнес/ниша**: Какой тип бизнеса? (фитнес, SaaS, инфопродукты, e-commerce, услуги, B2B)
2. **Цель экрана**: Что должен сделать посетитель? (оставить заявку, перейти в Telegram, скачать, купить)
3. **Целевая аудитория**: Кто будет смотреть? (возраст, боли, мотивация)

**Уточняющие вопросы (по необходимости):**

4. **Стиль дизайна**: Предпочтения? → см. references/styles.md
5. **Обязательные элементы**: Что включить? (видео-фон, форма, таймер, отзывы)
6. **Брендинг**: Есть ли цвета, шрифты, логотип?

### Шаг 2: Выбор стиля

После discovery выбери стиль из references/styles.md:
- **aggressive** — красно-чёрный, диагонали, пульсация (спорт, единоборства)
- **minimal** — много воздуха, один акцент (SaaS, tech)
- **corporate** — синие/серые тона, строгость (B2B, финансы)
- **creative** — градиенты, нестандартная геометрия (дизайн, маркетинг)
- **luxury** — тёмный фон, золото, анимации (премиум услуги)

### Шаг 3: Структура Hero

Стандартная структура первого экрана:

```
┌─────────────────────────────────────────────┐
│  [Logo]                    [Nav] [CTA btn]  │  ← Header
├─────────────────────────────────────────────┤
│                                             │
│           ГЛАВНЫЙ ЗАГОЛОВОК                 │  ← H1 (одна строка)
│         Подзаголовок с УТП                  │  ← Субтитл
│                                             │
│        [  CTA Button  ]                     │  ← Основной призыв
│        мелкий текст доверия                 │  ← Trust line
│                                             │
└─────────────────────────────────────────────┘
```

### Шаг 4: Создание кода

1. Создай `index.html` с семантической разметкой
2. Создай `styles/hero.css` с дизайн-токенами
3. Создай `js/hero.js` для анимаций (если нужны)

**Обязательные требования:**
- Mobile-first responsive
- Высота 100vh на десктопе
- Контрастный CTA (WCAG AA)
- Время загрузки < 3s

### Шаг 5: Проверка

После создания проверь:
- [ ] CTA виден без скролла
- [ ] Текст читается на фоне
- [ ] Работает на мобильных
- [ ] Анимации не мешают UX
